# ADR-005: Insights

> **Implementation status (2026-07-22):** Implemented.
> Insights are stored separately from raw events and generated by rule-based/declarative logic without replacing original event data. Remote declarative rules are accepted only after validation and verification against the fixed, expiring SHA-256 manifest described below.



OpenSecDash should not only display data.

It should detect relationships.


---

Example:

```none
Access Event
↓
404
```

plus

`Geoblock Event`

results in:

`404 probably caused by geoblock`


---

**Insights never replace original data.**

They are generated additionally.

## Remote rules authenticity

Remote rules and their manifest use fixed HTTPS paths. Before storing remote rules,
OpenSecDash validates the manifest path and expiry, verifies the exact ruleset bytes
with SHA-256, and then applies the existing schema and rule-count/field limits. A
network, expiry, hash, or validation failure preserves the last valid database rules
and the bundled fallback.

The SHA-256 manifest is an interim control through 2026-10-31. Because both files are
served from the same website, it does not independently protect against a compromise
of that website or its TLS delivery. Replacing it with an offline-key signature, or
renewing the exception with a documented scope and expiry, requires an explicit review.

Rules remain declarative JSON. Remote code, templates, dynamic imports, configurable
rule URLs, and executable content are not accepted.
